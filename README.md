# World Globe Study

スマホファーストの世界地図学習アプリです。`index.html` だけで動くので、ファイルを開くだけで試せます。GitHub Pages、Netlify、Cloudflare Pages などの静的ホスティングに置くと、そのままURLで共有できます。

## できること

- 地球儀をドラッグして回転
- ピンチまたはホイールで拡大縮小
- 国名・首都検索
- 国旗、国名、首都、地域、緯度経度を表示
- 今日の5か国
- 首都クイズ
- 2国の首都間距離を表示
- 選択中の国をURLハッシュに保存して共有

## ファイル構造

- `index.html`
  - HTML: 画面の部品
  - CSS: スマホ向けレイアウトと見た目
  - JavaScript: 地球儀の描画、国データ、検索、クイズ、距離計算

単体HTMLにしている理由は、共有・移動・バックアップが簡単だからです。外部ライブラリを使っていないため、通信環境に左右されにくい作りです。

## 重要な計算の考え方

### 1. 緯度経度を画面に変換する

地球儀は「正射投影」という考え方で描いています。正射投影は、地球を遠くから見たときのように、手前半分だけを丸く表示する方法です。

コードでは `project(lat, lon)` が担当しています。

- 緯度 `lat`: 北南の位置
- 経度 `lon`: 東西の位置
- `state.centerLon`: いま画面中央に見えている経度
- `state.centerLat`: いま画面中央に見えている緯度
- `z > 0`: 手前側なので描く
- `z <= 0`: 裏側なので描かない

この `z` 判定があるので、地球儀らしく「裏側の国が見えない」表現になります。

### 2. 大陸を3Dっぽく貼る

`LANDMASSES` に簡易的な大陸の形を入れています。最初に2Dの世界地図テクスチャを作り、その後で丸い地球の各ピクセルから緯度経度を逆算して色を取っています。

この処理は `renderGlobeTexture()` が担当しています。

### 3. 2国間距離

距離は `distanceKm(a, b)` で計算しています。使っているのはハーサイン式です。

地球を球として見たとき、2つの緯度経度の間の大円距離を求める式です。実際の地球は完全な球ではないため厳密値ではありませんが、学習用の目安として扱いやすい計算です。

## 国を追加する方法

`COUNTRIES` に1行追加します。

```js
{ cc: "XX", name: "国名", en: "English Name", capital: "首都名", lat: 0.00, lon: 0.00, region: "地域" }
```

- `cc`: 2文字の国コード。国旗表示に使います。
- `name`: 日本語名。
- `en`: 英語名。検索にも使います。
- `capital`: 首都名。
- `lat`: 首都の緯度。
- `lon`: 首都の経度。
- `region`: クイズや表示に使う地域名。

## もっと本格的にするなら

- 国境GeoJSONを読み込んで国境線を正確にする
- 人口、面積、言語、通貨を追加する
- 地域別クイズ、国旗クイズ、首都入力クイズを増やす
- 学習履歴を `localStorage` に保存する
- GitHub Pagesに置いてURL共有する
